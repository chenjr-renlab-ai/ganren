＊拓扑：一人架，多人连

ganren 平台是一个 Python 单进程服务。一个人在某台机器上跑起来，其他协作者本地的 Claude Code 通过 MCP 连过去。

   Alice 的 CC ──┐
                 │
   Bob 的 CC ────┼──→  ganren 平台 ──→ SQLite + Slack
                 │
   Carol 的 CC ──┘

1架平台（管理员，做一次）

装依赖 + 启动

git clone https://github.com/chenjr-renlab-ai/ganren.git
cd ganren
uv sync --extra dev
cp .env.example .env
uv run python -m ganren_platform.main

看到 Uvicorn running on http://0.0.0.0:8787 即就绪。

让队友连进来 —— 选一种

场景方案队友填的 URL

跨网临时ngrok http 8787https://xxx.ngrok-free.app/mcp/

Slack 通知（可选但强推）

在 .env 填 SLACK_WEBHOOK_URL，重启平台。所有任务事件（发布 / 接走 / 提问 / 回复 / 提交 / 关闭）自动广播到对应频道。

2每个协作者（一次性）

装 Claude Code + 配 MCP

在项目根目录或 ~/.claude.json 里加：

{
  "mcpServers": {
    "ganren": {
      "type": "http",
      "url": "<管理员给的 URL>"
    }
  }
}

告诉 CC 你是谁（关键步骤）

在 ~/.claude/CLAUDE.md 里加一行：

# ganren 平台协作
我在 ganren 上的 actor handle 是 `bob`。
调任何 ganren__* 工具时，actor 都用 bob。

这样 CC 调任何 ganren 工具会自动带上你的 handle。

试连

跟 CC 说："看下我在 ganren 上的 inbox"，第一次会弹权限提示，确认即可。

你跟 CC 说自然语言，CC 自动调对应工具。

场景你跟 CC 说CC 自动调

4几个关键设计选择

list_open_tasks 看不到完整 context · 协作者必须 claim 后才能看 context_summary 和 artifacts。防止"看一眼就走"导致的信息泄露和决策失真。
rework_count 挂任务不挂人 · 被打回不扣个人分；rework 累计在任务上，目的是鼓励反复打磨而不是惩罚返工。
hard / DRI 任务强制填 decision_record · 包含选项、最终选择、估计成功概率、原因。考核框架按"下注质量"考核 DRI，而不是结果好坏，避免坏运气罚掉做对决策的人。
求助每次记一笔，不扣分 · 但下游考核仪表盘聚合时，逐周向外求助数减少 = IC 货币的强加分项（"你在变自主"）。
平台只出原始事件 + 聚合查询 · 不计算合成分、不暴露"开除"动作。地板灯由下游仪表盘按绝对地板算。诊断不裁决。
